How Sure Were You?

We knew the expected probabilities of all three coins shown in the guessing games you just played. However, it is very unlikely that as you flipped the results lined up exactly with how the probabilities were set up. Why? Because of random chance.

Probability Distributions and the Issue with Random Chance

In statistics, probability distributions are mathimatical functions which describe the probability of different outcomes of an experiment. Take our first coin example; the two possible outcomes are the two sides and the probability of each side on any given flip is set - 50% for the fair coin and 75% for the weighted one.

However, this probability of results gets in the way of accurately understanding real world data. It was likely that at most points of your experiment with the second coin game that the number of blue and purple results were not the same. But how do you know if it was due to it being weighted, or to simple chance?

The normal distribution is one of many different distributions out there. In fact, with real world data and experiments, there is no mathimatical model that fits the data perfectly. So how do we find the true value of data? How do we know if the coin is truly weighted?

Well, we can’t. Not with 100% certainty anyways. But we can make a good guess if we design a good experiment. Thats where the Central Limit Theorem comes in! Wolfram MathWorld defines the Central Limit Theorem as:

To explain it more simply so we can get to discussing error types, what the CLT basically tells us is this: For anything we take independent samples of, as long as we take enough samples (usually meaning over 30), the the sample mean approaches the true mean, with a standard deviation of σn\frac{\sigma}{\sqrt{n}}n​σ​ where n is the sample size and and σ\sigmaσ is the population standard deviation. By taking advantage of this fact, we can predict the true characteristics of the population without needing to know or model the true population.

So Sampling Enough Times Answers Everything?

Nope. Unfortunately, randomness still gets in the way. Even if we take an infinite number of samples, it will still be a normal distribution, which goes to infinity. Unfortunately, statistics in the real world, like most things, is never certain. Furthermore, testing is often expensive and sample sizes are often limited due to resources. The best we can do is hedge our prediction. That is where hypothesis testing comes in!

Hypothesis Testing

In this explorable explanation we will be focusing on how a Z-test hypothesis test works.

A Z-test is a type of statistical hypothesis test that uses the CLT to “reject” or “fail to reject” a null hypothesis ( HoH_oHo​ ). A null hypothesis is the assumption that nothing is different, or that the result is not statisticly significant from the accepted population. For example, the null for the coin games was that the coin was fair. The null for a new medicine would be that it does not do anything.

The alternative hypothesis ( HaH_aHa​ ) is contrary to the null hypothesis. It is the effect we are looking for in the test. For example, in the second coin game, the alternative hypothesis is that the probability of each possible result is not 1/2. The alternative hypothesis can either be right sided ( Ho<HaH_o < H_aHo​<Ha​ ), left sided ( Ho>HaH_o > H_aHo​>Ha​ ), or two sided ( Ho<>HaH_o <> H_aHo​<>Ha​ ) depending on the effect we are testing for.

P-Values and Confidence

Note the language here; we always give a confidence level, and if the p-value is not less than alpha we fail to reject the null instead of accepting it. This is because the p-value is a probability. For example, if we set alpha to 0.05, we would reject the null if we got a p-value of 0.03. However, 0.03 means that there is a 3% chance we saw this sample mean given the population mean and standard deviation. Similarly, if we got a p-value of 0.07, we would fail to reject the null because it is not past the threshold we set of 0.05, but we cannot accept the null as we do not have evidence that it is true, we just don’t have enough that it is false.

With the CLT, we can model the distribution of or sample mean as N[μ,σn]N[\mu, \frac{\sigma}{\sqrt{n}}]N[μ,n​σ​] . This distribution tells us the expected results of the null hypothesis. The other components of a statistical hypothesis test are the p-value and the significance level ( α\alphaα ). The p-value tells us the probability we got the sample mean we observed given the null distribution. We determine α\alphaα beforehand to decide how robust we want our test to be. If the p-value is less than alpha, we reject the null at a confidence level of 1-alpha. If it is not less than alpha we fail to reject the null.

Error

As with any statistics that deal with probability, we also have to take error into account. There are two types of error in a hypothesis test; type I and type II.

Type I Error

A type I error occurs when we reject the null hypothesis but it is actually true. This can be thought of as a false positive. The probability of a type I error is α\alphaα . Recall that we set α\alphaα ourselves. The α\alphaα value we pick is the risk we are willing to take when we reject the null that the data we saw was due to the variability of the null distribution. This is the same as a type I error.

Type II Error

A type II error occurs when we fail to reject the null when it is false. This is the same as a false negative. The probability of a type II error is β\betaβ and cannot be calculated when the HaH_aHa​ is greater than, less than, or not equal to HoH_oHo​ like we saw above. However, we can calculate it if we have a HaH_aHa​ we are testing for.

Importance of Choosing the Right Significance Level

As you can see from the graph above, unless the means of the HoH_oHo​ and HaH_aHa​ are far apart or their standard deviations are very low (which is unlikely in a real world test), changing the significance level to reduce type I error will always increase the chance of a type II error and vice versa. Thus, we should always keep in mind the impact of a false positive vs a false negative and design the test so that the worst case outcome is minimized. Take two real world examples: a criminal trial and a clinical trial for a new screening for a dangerous medical condition.

In the criminal trial case, innocent until proven guilty is considered a human right by the UN’s Universal Declaration of Human Rights, so it is much worse to have a type I error than a type II. Thus, we set the significance level very low, which is why in the US federal court juries much reach a unanimous verdict. In the medical screening case, we would much rather have a type I error with a false positive which can be corrected than a missed diagnosis where someone could die because they thought they didn’t have the condition. Thus, the significance level would be set higher.

Z-test Issues

Z-tests are not without downsides. Even with a standard significance level of 0.05, 1/20 tests that should fail to reject the null will reject the null due to random error. Also, since theoretically you could continously resample until you get the result you are looking for, Z-tests can only be done once as repeated tries are considered unethical. This causes a problem, as a core foundation of science is that experiments should be reproducable.
